Artifact 01

Chop Chop

A conversational AI meal-planning assistant.

Introduction

Chop Chop is a conversational AI assistant that plans a meal from the ingredients on hand, the number of people being fed, any dietary needs, and the time available to cook. It gathers missing details one question at a time rather than through a form, then returns a complete recipe: ingredient quantities scaled to the number of people, a full numbered method, prep and cook time, and a suggestion for what to serve alongside.

Objective

To design a conversational AI system from end to end: define its behaviour, test it against a range of realistic and adversarial inputs, and produce documented evidence that it performs as specified, rather than relying on a single successful demonstration.

Process

I began by iterating on the system prompt until Chop Chop consistently produced complete recipes rather than partial outlines, and asked a single clarifying question when information was missing instead of guessing at an answer.

I then built in explicit guardrails. A declared allergy is treated as a strict constraint rather than a preference. Requests for medical or clinical nutrition advice are declined, with the user directed to a professional instead. Off-topic requests, such as being asked to write an essay, are declined and redirected back to food.

I tested the system against a set of scenarios covering both core function and guardrail behaviour: a full recipe generated from a short ingredient list, a dietary substitution, a nut allergy, a deliberately vague prompt ("I'm hungry"), a request for medical advice, and an off-topic request. Each interaction was captured as a screenshot.

Tools

Claude, accessed via Cowork, for the build and iterative prompt design. Microsoft Word for the evidence documentation.

Value

This is a modest project, a meal planner rather than a commercial system, but the underlying skill is the one that transfers to more serious work: taking a loosely defined need and turning it into a system that behaves predictably, including at points where it would be easiest to improvise.

The guardrail behaviour is the part most worth noting. Producing a recipe from a model is straightforward. Deciding what the system must refuse to do, and demonstrating that it consistently does refuse, is what separates something suitable for a real user from something that merely performs well in a demonstration.

Evidence

Chop Chop runs in a Cowork sidebar rather than on a public server, so this is tested behaviour rather than a live link.
